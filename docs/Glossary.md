# Glossary (Updated)

## Unified CX Intelligence System — Glossary of Terms (Updated with Reporting Layer)

This glossary defines all key terms, concepts, workflows, and components used in the Unified CX Intelligence System, including the newly added **Reporting & Analytics Layer**, **Weekly Executive Summary workflow**, and dashboard elements.

---

# A

### **Alert (EWS Alert)**
A high‑severity notification sent to Slack for human review when Diagnostics identifies a risky event.

### **Asana Workflow**
The set of n8n automations responsible for task creation, content review, and final publishing confirmation.

---

# C

### **CMO Metrics (NEW)**
Metrics intended for leadership-level reporting, including sentiment trends, severity distribution, response times, and system health.

### **Content Review Workflow**
Workflow 05 responsible for monitoring task status and communicating updates to Slack.

### **Critical Event**
An event with incident score typically ≥ 8.5, requiring immediate human attention.

---

# D

### **Dashboard (NEW)**
A visualization layer built on Google Sheets/BI tools using the system logs to provide:
- Volume trends  
- Severity distribution  
- Sentiment evolution  
- Risk patterns  
- Team performance  
- System health  

### **Dataflow**
A representation of how data moves through the system from ingestion to reporting.

### **Diagnostics**
Workflow 02 where sentiment analysis, severity scoring, topic extraction, and recurrence analysis take place.

---

# E

### **Executive Summary (NEW)**
A weekly leadership-focused report generated by Workflow 07, summarizing event volume, risk, sentiment, trends, and system health.

### **EWS (Early Warning System)**
The system that detects high-risk events and sends alerts to Slack for human review.

---

# G

### **Governance**
The system’s strict policies ensuring:
- Zero auto-publishing  
- Human-in-the-loop decisions  
- Read-only platform access  
- Full audit logging  
- Credential safety  

---

# H

### **HealthCheck**
Workflow 00 verifying connectivity, API validity, and system uptime.

### **Human-in-the-Loop**
A mandatory requirement that all high‑risk or externally-impacting decisions are approved by a human operator via Slack.

---

# I

### **Incident Score**
A numerical value (0–10) representing risk severity based on sentiment, recurrence, impact, spread potential, and platform weight.

### **Ingestion Layer**
The system layer responsible for fetching and normalizing data from external platforms.

---

# L

### **Logging Engine**
Workflow 01 that writes logs to Google Sheets including workflow outputs, errors, severity scores, and human decisions.

### **Log Sheet**
The Google Sheets table used as persistent storage for all system activity and as the source for dashboards and weekly reports.

---

# M

### **Metrics Framework (NEW)**
A document defining:
- System performance metrics  
- Risk/incident metrics  
- Sentiment metrics  
- Team responsiveness metrics  
- Health and governance metrics  
Used by leadership and dashboard builders.

---

# P

### **Platform Weight**
A numerical modifier used in incident scoring to adjust severity based on a platform’s reputational impact and reach.

### **Publishing Confirmation**
Workflow 06 ensuring content approval and safety checks before manual human publishing.

---

# R

### **Recurrence Level**
A scoring factor representing how frequently similar complaints or events appear across time.

### **Reporting Layer (NEW)**
A new architectural component responsible for:
- Weekly Executive Summary  
- Dashboard analytics  
- Metrics documentation  

Transforms raw logs into executive intelligence.

---

# S

### **Sentiment**
The calculated emotional tone of an inbound message (positive, neutral, negative).

### **Severity Tier**
Classification of incident scores:
- Low  
- Mild  
- Moderate  
- High  
- Critical  

### **System Overview**
A high-level document summarizing architecture, layers, workflows, and principles.

---

# T

### **Topic Extraction**
A diagnostic process that identifies common categories or themes across multiple events.

### **Trend Analysis**
The study of changes in volume, severity, sentiment, or topics over time.

---

# W

### **Weekly Report (NEW)**
A weekly executive-level summary generated by Workflow 07 and delivered via Slack. Includes:
- Volume metrics  
- Severity distribution  
- Sentiment trend  
- Escalations  
- Review speeds  
- System health indicators  

### **Workflow**
A modular n8n automation performing a specific role in the system.

---

# Summary

The updated glossary reflects all components of the system, including the new Reporting & Analytics Layer, executive summary workflow, dashboard concepts, and expanded metrics terminology.
